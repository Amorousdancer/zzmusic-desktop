# Codex 工作说明

## 项目名称

ZZmusic

## 工作原则

- 不一次性做太多功能，按阶段稳定推进。
- 每次开发优先保证应用可运行。
- 新功能先满足第一版需求，再考虑扩展。
- 不提前引入复杂依赖、数据库或过度抽象。
- 涉及用户文件的操作必须保守：移除播放列表项不能删除原文件。

## 标准文件路径

- 项目需求：`docs/01-项目需求.md`
- 技术方案：`docs/02-技术方案.md`
- 设计规范：`docs/03-设计规范.md`
- 执行步骤：`docs/04-执行步骤.md`
- 验收标准：`docs/05-验收标准.md`
- 开发日志说明：`开发日志/README.md`
- 当天开发日志：`开发日志/YYYY-MM-DD.md`

## 每次开发流程

1. 先阅读 `Codex.md` 和相关 `docs/` 标准文件。
2. 明确本次只处理哪个阶段或哪个小功能。
3. 修改前检查现有文件，避免覆盖用户已有改动。
4. 完成后进行最小必要验证。
5. 更新当天开发日志的已完成事项和待办事项。
6. 向用户汇报本次完成内容、验证结果和下一步建议。

## 当前推进顺序

1. 搭建 Electron + React + TypeScript + Vite 最小可运行架构。
2. 实现本地音乐导入和播放列表持久化。
3. 实现播放控制、进度条和音量。
4. 完成 Apple Music 风格浅色界面。
5. 准备 Windows x64 打包。
