# 技术方案

## 推荐技术栈

第一版建议使用：

- Electron：提供 Windows 桌面壳、文件选择、本地文件访问能力。
- React：构建播放器界面。
- TypeScript：降低后期扩展时的状态和接口风险。
- Vite：提供轻量开发服务和前端构建。

## 架构原则

- 先做最小可运行版本，再补功能。
- 播放能力优先使用浏览器原生 `HTMLAudioElement`。
- 本地保存优先使用简单 JSON 文件，不提前引入数据库。
- 主进程只处理系统能力：窗口、文件选择、读写本地配置。
- 渲染进程只处理界面、播放状态和交互。
- 主进程与渲染进程通过受控 IPC 通信。

## 数据保存

第一版保存内容：

- 歌曲唯一标识。
- 本地文件路径。
- 文件名。
- 导入时间。
- 可选：歌曲时长。

保存位置：

- 使用 Electron 的 `app.getPath("userData")` 目录。
- 文件名建议：`library.json`。

## 安全约束

- Electron 窗口启用 `contextIsolation`。
- 默认关闭 `nodeIntegration`。
- 通过 preload 暴露最小 API。
- 导入音乐时只接受明确允许的音频扩展名。
- 移除歌曲只修改播放列表，不删除本地文件。

## 打包目标

第一阶段先保证开发环境可运行。

后续再加入：

- Windows x64 打包。
- 安装包或绿色版产物。
- 应用图标和版本信息。
